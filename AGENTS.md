# Repository Guidelines

## Project Structure & Module Organization
- `src/` holds the TypeScript backend (Express app, services, routes, config).
  - `src/controllers/` REST controllers.
  - `src/services/` scheduling and optimization logic.
  - `src/types/` shared domain models and interfaces.
- `tests/` contains Jest unit tests (`*.test.ts`).
- `client/` and `public/` hold UI assets (if you touch UI, keep API and UI changes aligned).
- `prisma/` includes schema and migrations; `scripts/` has one-off maintenance scripts.
- `dist/` is build output (generated by `npm run build`).

## Build, Test, and Development Commands
- `npm run dev` starts the API with nodemon (TS entry: `src/server.ts`).
- `npm run build` compiles TS to `dist/`.
- `npm start` runs the built server from `dist/server.js`.
- `npm test` runs Jest with `ts-jest`.
- `npm run clean` clears `dist/`.
- Prisma helpers: `npm run db:generate`, `npm run db:push`, `npm run db:migrate`, `npm run db:studio`.

## Coding Style & Naming Conventions
- TypeScript (CommonJS) with `strict` enabled (`tsconfig.json`).
- Indentation is 4 spaces; keep imports sorted and grouped by module type.
- Naming: `PascalCase` for types/classes, `camelCase` for variables/functions, `kebab-case` for file names where applicable (e.g., routes).
- No formatter or linter is configured; keep changes minimal and consistent with nearby code.

## Testing Guidelines
- Framework: Jest with `ts-jest`.
- Test files use `*.test.ts` and live in `tests/` or alongside source under `src/`.
- Prefer focused unit tests for scheduling logic in `src/services/`.

## Commit & Pull Request Guidelines
- Recent commits use imperative messages; some include a type prefix like `feat:`. Follow that pattern when possible.
- PRs should include: a short summary, test results (command + outcome), and any relevant API or UI notes.
- If a change affects UI (`client/` or `public/`), add screenshots or a brief GIF.

## Configuration Tips
- Environment is driven by `.env` (see config in `src/config/`); avoid hardcoding secrets.
- Database changes should be reflected in Prisma schema and migrations before merging.
